What is ResearchStack?

ResearchStack is an SDK and UX framework for building research study apps on Android.

Is it similar to iOS ResearchKit(TM)?

Yes, an overriding goal of ResearchStack is to help developers and researchers with existing apps on iOS more easily adapt those apps for Android. Though the correspondence of features between the two SDKs isn’t one-to-one, the two SDKs will offer enough shared functionality and a common framework and naming scheme to greatly speed up adaptation of ResearchKit apps to Android (and ResearchStack apps to iOS) and the procedural aspects of running a study on a new platform (such as IRB approval and secure connectivity with a data collection backend).

How can I get started? Where is all the code?

In an effort to make basic functionality available to researchers as quickly as possible, we are planning a phased rollout of ResearchStack.

On Friday 2/19/15 our Beta Preview early access program begins. Researchers and developers who would like to view the status of ResearchStack and follow along with the progress of the first phase are encouraged to request an invite using the signup sheet provided above

Research Stack 1.0 will be publicly available for use and contributions by everyone by April 15, 2016.

Beta Preview access

The early access phase is targeted at researchers and developers who want to get a head start building their Android apps by understanding what will be included in version 1.0. While we won’t be directly accepting pull requests the ResearchStack core development team will be reviewing feedback and reaching out to people to ensure that the release is stable and production-ready.

ResearchStack 1.0 release

ResearchStack 1.0 will be publicly available on April 15th, 2016.

Researchers can expect the ability to build a basic Android app without having extensive knowledge of the Android platform. Using JSON and HTML files a simple app can be built that will include the onboarding flow (Study Overview/Consent/Sign Up) and an Activities screen with scheduled tasks for the user. Researchers will also be able to reuse existing ResearchKit AppCore resources, with some minor changes.

Included graphing libraries will facilitate data visualization. ResearchStack 1.0 will also have tools to allow easy on-disk encryption and strives to be extensible so that researchers can implement or modify screens they need.

An open source Android version of Mole Mapper will be released alongside ResearchStack 1.0 as an example implementation of the platform.

Who is backing this project?

ResearchStack is developed by Cornell Tech's Small Data Lab and Open mHealth in collaboration with touchlab. Initial funding for the project is provided by the Robert Wood Johnson Foundation.
